Atlas Bank - Gestion Finance iOS

Démonstration de l'interface utilisateur et du mode discrétion.

Objectif : Concevoir une application mobile native performante pour le suivi des finances personnelles avec une attention particulière à l'UX/UI et la sécurité des données.

Technologies :
  • SwiftUI : Interface utilisateur moderne et réactive.
  • SwiftData : Persistance locale des données (CRUD Utilisateurs, Transactions).
  • Swift Charts : Visualisation graphique des dépenses.
  • MVVM : Architecture propre séparant la vue de la logique métier.
Défis techniques & Solutions :

Le principal défi a été l'adoption de SwiftData, une technologie très récente d'Apple. J'ai dû apprendre à gérer les migrations de schémas et les relations complexes entre entités (Utilisateurs <-> Transactions) sans documentation abondante. L'implémentation du "Mode Discrétion" a aussi nécessité une gestion fine des états globaux de l'application (@EnvironmentObject).

Fonctionnalités clés :
  • Authentification sécurisée et gestion de profil (Cartes Gold/Standard).
  • Dashboard interactif avec calcul du "Reste à vivre" en temps réel.
  • Mode Discrétion : Floutage des montants sensibles d'un simple switch.
  • Alertes budgétaires intelligentes basées sur des seuils personnalisables.

Games Room - Arcade JavaScript

Objectif : Développer un moteur de jeu web léger sans utiliser de frameworks externes, pour maîtriser les bases du DOM et du Canvas HTML5.

Technologies :
  • HTML5 / CSS3 (Grid & Flexbox pour le layout responsive).
  • Vanilla JavaScript (ES6+) : Logique pure sans librairie.
  • Canvas API : Rendu graphique fluide pour le jeu Snake.
Défis techniques & Solutions :

La réalisation de l'IA pour le Morpion a été un défi algorithmique majeur. J'ai dû implémenter l'algorithme Minimax pour rendre l'ordinateur invincible. Pour le Snake, la gestion de la boucle de jeu (Game Loop) et du rafraîchissement via requestAnimationFrame a été essentielle pour éviter les saccades.

Jeux développés :
  • Snake : Gestion des collisions et boucle de jeu performante sur Canvas.
  • Morpion (Tic-Tac-Toe) : Mode 2 joueurs ou contre une IA adaptative.
  • Pendu : Manipulation dynamique du DOM (création de boutons, affichage du bonhomme) et gestion des états de victoire/défaite.

QuanticGP - Dashboard Prestataires

Objectif : Digitaliser la gestion des prestataires pour le Grand Prix de Monaco, facilitant la réservation et l'attribution des stands via une plateforme web complète.

Technologies :
  • Express.js (Backend API REST)
  • PostgreSQL (Base de données relationnelle)
  • Vue.js (Frontend réactif)
  • Node.js (Environnement serveur)
Mon rôle dans l'équipe (5 pers.) :

En tant que développeur Full-Stack au sein de l'équipe Agile, j'ai pris en charge la conception de l'API REST pour la gestion des activités et des stands. J'ai également travaillé sur l'intégration Frontend (Vue.js) pour assurer une communication fluide avec le serveur et une expérience utilisateur dynamique.

Défis techniques & Solutions :

La gestion de la concurrence d'accès sur les réservations a été critique (éviter que deux prestataires réservent le même stand simultanément). Nous avons résolu cela grâce à une gestion rigoureuse des transactions SQL (ACID) et des verrouillages optimistes au niveau de la base de données PostgreSQL.

ScreenSerenity (iOS)

Objectif : Aider les utilisateurs à réduire leur addiction au smartphone via des statistiques claires, un suivi d'objectifs et des outils de restriction douce.

Technologies :
  • Swift / SwiftUI
  • ScreenTime API (DeviceActivity)

Travailler avec l'API ScreenTime d'Apple est complexe car elle est très restrictive pour des raisons de vie privée. J'ai dû apprendre à utiliser le framework DeviceActivity pour récupérer les données d'utilisation de manière anonymisée et les afficher sous forme de graphiques compréhensibles pour l'utilisateur sans compromettre la sécurité.

Fonctionnalités :
  • Visualisation graphique du temps d'écran quotidien/hebdomadaire.
  • Définition d'objectifs de déconnexion par application (ex: 30min de TikTok max).

4Mation - Jeu de Stratégie

Objectif : Développer un jeu de plateau complet (variante du Puissance 4 avec rotation) en respectant strictement le pattern MVC (Modèle-Vue-Contrôleur) en Java.

Technologies :
  • Java (Swing pour l'interface graphique)
  • Junit (Tests unitaires)
  • Git (Gestion de versions en équipe)
Mon rôle dans l'équipe :

Développeur principal sur la couche Modèle. J'ai été responsable de l'implémentation des règles du jeu, de la gestion de la grille de données et de la validation des mouvements. J'ai également assuré la coordination avec la vue via le pattern Observer.

Défis techniques & Solutions :

Le plus grand défi a été de séparer proprement l'affichage (Vue) de la logique métier (Modèle). Nous avions tendance au début à mélanger le code. Nous avons résolu cela en appliquant strictement le pattern Observer : le modèle notifie la vue des changements sans jamais la connaître directement, garantissant un code modulaire et testable.

NFT Universe

Objectif : Créer une vitrine e-commerce moderne et responsive pour une collection fictive de NFT, avec gestion de panier côté client.

Technologies :
  • HTML5 / CSS3
  • Bootstrap 5 (Mise en page responsive)
  • JavaScript (Manipulation du DOM & LocalStorage)
Mon rôle dans l'équipe :

J'ai travaillé sur l'intégration Frontend et la logique JavaScript du panier. Ma mission était de rendre le site parfaitement utilisable sur mobile (Responsive Design) et de persister le choix des articles de l'utilisateur même après rechargement de la page.

Défis techniques & Solutions :

La gestion de l'état du panier sans Backend a été le point technique central. J'ai utilisé l'API LocalStorage du navigateur pour sauvegarder les données du panier (ajout, suppression, calcul du total) afin de simuler une expérience e-commerce persistante sans base de données réelle.

Fonctionnalités :
  • Galerie filtrable de produits NFT.
  • Panier dynamique avec calcul automatique du total.
  • Design totalement responsive adapté aux mobiles et tablettes.
